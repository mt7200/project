中医医疗系统开发计划 (每日自测 + 分阶段联调)

📌 项目技术规范摘要：
技术栈
前端:Vue 3.5 + TypeScript 5.6 + Vite 6 + Vue Router 4 + ECharts 5.5
后端:Python 3.10+ / FastAPI 最新 / SQLAlchemy 2.x / MySQL 8.x InnoDB
IDE:VS Code (前端) + PyCharm (后端)
核心业务约束
剂量校验dosage ≥ min_dosage & dosage ≤ max_dosage
配伍禁忌十八反/十九畏检测 (甘草 - 甘遂/乌头 - 半夏等)
审核状态draft→pending→approved/rejected
团队分工
李鮋数据库+核心 CRUD APIherb, herb_incompatibility, formula, formula_herb, patient, visit_record, emr, syndrome_pattern, symptom_dict; /api/patients/*, /api/visits/*, /api/herbs/*, /api/formulas/*, /api/emrs/*, /api/symptom-dict/*, /api/syndrome-patterns/* API
沈文博业务 API+ 服务层auth/login, prescription, prescription_review, statistics, dosage_check_service, incompatibility_check_service, risk_evaluation_service; /api/auth/*, /api/prescriptions/*, /api/review/*, /api/statistics/*
罗方程所有前端页面/API 封装需要完整 API 文档和 Swagger 地址
开发模式
前端mock 先跑 UI → 当天接口联调 → Bug 当日清零
后端Swagger/Postman 自测通过 → 提供开放接口 → 支持 CORS 跨域
工作时间
每日时段上午 9:00-12:00 | 下午 13:30-18:00 (7.5h/天)
