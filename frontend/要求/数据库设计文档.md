## 四、数据库表设计

### 4.1 sys_user — 系统用户表

对应前端：`Login.vue` 中 `LoginRequest` / `User` 接口

```sql
CREATE TABLE sys_user (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    username        VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password_hash   TEXT         NOT NULL COMMENT '密码哈希值',
    role            VARCHAR(20)  DEFAULT 'doctor' COMMENT '角色：admin-管理员，doctor-医生',
    real_name       VARCHAR(50)  COMMENT '真实姓名',
    phone           VARCHAR(20)  COMMENT '联系电话',
    is_active       TINYINT      DEFAULT 1 COMMENT '是否启用：0-禁用，1-启用',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) COMMENT '系统用户表';
```

### 4.2 patient — 患者信息表

对应前端：`PatientEntry.vue` 中 `PatientForm` / `Patient` 接口

```sql
CREATE TABLE patient (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    name            VARCHAR(50)  NOT NULL COMMENT '姓名',
    gender          VARCHAR(10)  COMMENT '性别：男/女',
    age             INT          COMMENT '年龄',
    phone           VARCHAR(20)  UNIQUE COMMENT '联系电话',
    id_card         VARCHAR(20)  UNIQUE COMMENT '身份证号',
    address         VARCHAR(200) COMMENT '住址',
    blood_type      VARCHAR(10)  COMMENT '血型',
    height          DECIMAL(4,1) COMMENT '身高 (cm)',
    weight          DECIMAL(5,1) COMMENT '体重 (kg)',
    allergy_info    TEXT         COMMENT '过敏史',
    status          VARCHAR(20)  DEFAULT 'active' COMMENT '状态：active-在院，discharged-出院，deceased-去世',
    created_by      BIGINT       COMMENT '建档人 (sys_user.id)',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (created_by) REFERENCES sys_user(id)
) COMMENT '患者信息表';
```

### 4.3 visit_record — 就诊记录表（含四诊信息）

对应前端：`Workspace.vue` 中 `VisitRecord` / `ChiefComplaintTag` / `PresentIllnessTag` 等接口

```sql
CREATE TABLE visit_record (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    patient_id      BIGINT       NOT NULL COMMENT '患者 ID',
    doctor_id       BIGINT       NOT NULL COMMENT '主治医生 ID',
    visit_date      DATE         NOT NULL COMMENT '就诊日期',
    chief_complaint TEXT         COMMENT '主诉（症状标签组合）',
    present_illness TEXT         COMMENT '现病史（标签 + 描述）',
    past_history    TEXT         COMMENT '既往史',
    allergy_history TEXT         COMMENT '过敏史',
    personal_history TEXT        COMMENT '个人史',
    tongue_image    TEXT         COMMENT '舌象照片路径/描述',
    pulse_image     TEXT         COMMENT '脉象照片/描述',
    other_exams     TEXT         COMMENT '其他检查（辅助检查）',
    status          VARCHAR(20)  DEFAULT 'ongoing' COMMENT '状态：ongoing-进行中，completed-已完成',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (patient_id) REFERENCES patient(id),
    FOREIGN KEY (doctor_id)  REFERENCES sys_user(id),
    INDEX idx_patient_visit (patient_id, visit_date)
) COMMENT '就诊记录表（含四诊信息）';
```

### 4.4 diagnosis_result — 辨证诊断结果表

对应前端：`DiagnosisFlow.vue` 中 `DiagnosisResult` 和 `CustomDiagnosis` 接口

```sql
CREATE TABLE diagnosis_result (
    id            BIGINT PRIMARY KEY AUTO_INCREMENT,
    visit_id      BIGINT       NOT NULL COMMENT '就诊记录 ID',
    pattern_name  VARCHAR(100) NOT NULL COMMENT '证型名称，如：肝阳上亢证',
    pattern_code  VARCHAR(20)  COMMENT '证候编码，如：B01.02',
    confidence    INT          COMMENT '置信度（0-100），AI 诊断时使用',
    description   TEXT         COMMENT '证型描述/病机分析',
    symptoms      TEXT         COMMENT '关键症状（JSON 数组或逗号分隔）',
    etiology      VARCHAR(200) COMMENT '病因',
    pathogenesis  VARCHAR(200) COMMENT '病机',
    is_selected   TINYINT      DEFAULT 0 COMMENT '是否为最终选定诊断：0-否，1-是',
    source        VARCHAR(20)  DEFAULT 'manual' COMMENT '来源：ai-AI 诊断，manual-手动选择',
    created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (visit_id) REFERENCES visit_record(id),
    INDEX idx_visit (visit_id)
) COMMENT '辨证诊断结果表';
```

### 4.5 herb — 中药材表

对应前端：`PrescriptionFlow.vue` / `IntelligentPrescription.vue` 中 `Herb` 接口

```sql
CREATE TABLE herb (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    name        VARCHAR(50)  NOT NULL UNIQUE COMMENT '药材名称',
    category    VARCHAR(50)  NOT NULL COMMENT '分类：解表药/清热药/补虚药/化痰药等',
    nature      VARCHAR(20)  COMMENT '药性：寒/微寒/温/大热/平',
    taste       VARCHAR(20)  COMMENT '药味：甘/苦/辛/酸/咸等',
    meridian    VARCHAR(100) COMMENT '归经：如 肝、胆',
    min_dosage  DECIMAL(5,1) COMMENT '药典最小剂量',
    max_dosage  DECIMAL(5,1) COMMENT '药典最大剂量',
    unit        VARCHAR(5)   DEFAULT 'g' COMMENT '单位',
    is_toxic    TINYINT      DEFAULT 0 COMMENT '是否有毒：0-否，1-是',
    functions   TEXT         COMMENT '功效（JSON 数组），如：["疏散退热","疏肝解郁"]',
    indications TEXT         COMMENT '主治',
    created_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category),
    INDEX idx_nature (nature)
) COMMENT '中药材表';
```

### 4.6 herb_incompatibility — 配伍禁忌关系表

对应前端：`Herb.incompatibilities` 和 `Herb.synergies` 字段

```sql
CREATE TABLE herb_incompatibility (
    id         BIGINT PRIMARY KEY AUTO_INCREMENT,
    herb_a_id  BIGINT      NOT NULL COMMENT '药材 A 的 ID',
    herb_b_id  BIGINT      NOT NULL COMMENT '药材 B 的 ID',
    type       VARCHAR(20) NOT NULL COMMENT '关系类型：incompatibility-十八反/十九畏，synergy-相须相使',
    detail     VARCHAR(200) COMMENT '详细说明',
    FOREIGN KEY (herb_a_id) REFERENCES herb(id),
    FOREIGN KEY (herb_b_id) REFERENCES herb(id),
    UNIQUE KEY uk_herb_pair (herb_a_id, herb_b_id, type)
) COMMENT '配伍禁忌与配伍关系表';
```

### 4.7 formula — 方剂表

对应前端：`commonFormulas` 数据；数据来源：`标注二级分类后的方剂表.xlsx`

```sql
CREATE TABLE formula (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    name            VARCHAR(100) NOT NULL UNIQUE COMMENT '方剂名称',
    category_l1     VARCHAR(50)  COMMENT '一级方剂类型，如：经典方剂',
    category_l2     VARCHAR(50)  COMMENT '二级方剂类型，如：固涩剂/安神剂/补益剂等',
    functions       TEXT         COMMENT '功能，如：敛肺止咳，益气养阴',
    indications     TEXT         COMMENT '主治，如：久咳伤肺，气阴两伤证',
    modifications   TEXT         COMMENT '加减化裁，如：若虚热明显，可加地骨皮、麦冬',
    source          VARCHAR(200) COMMENT '来源/出处',
    prescription_id VARCHAR(50)  COMMENT '处方 ID（Excel 原始 ID，用于数据追溯）',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_category (category_l1, category_l2)
) COMMENT '方剂表（数据来源：标注二级分类后的方剂表.xlsx）';
```

### 4.8 formula_herb — 方剂组成表（方剂 - 药材关联）

对应前端：`commonFormulas.herbs` 数组；数据来源：Excel 方剂表中每行药材记录

```sql
CREATE TABLE formula_herb (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    formula_id  BIGINT       NOT NULL COMMENT '方剂 ID',
    herb_id     BIGINT       NOT NULL COMMENT '药材 ID',
    herb_name   VARCHAR(50)  COMMENT '药材名称（冗余，便于查询）',
    dosage      VARCHAR(20)  COMMENT '参考剂量，如：12g',
    amount      VARCHAR(20)  COMMENT '含量（Excel 原始字段）',
    sort_order  INT          DEFAULT 0 COMMENT '排序序号',
    FOREIGN KEY (formula_id) REFERENCES formula(id),
    FOREIGN KEY (herb_id)    REFERENCES herb(id),
    UNIQUE KEY uk_formula_herb (formula_id, herb_id)
) COMMENT '方剂组成表（方剂 - 药材多对多关联）';
```

### 4.9 prescription — 处方表

对应前端：`PrescriptionFlow.vue` 中 `Prescription` 接口

```sql
CREATE TABLE prescription (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    prescription_no VARCHAR(50)  UNIQUE COMMENT '处方编号',
    patient_id      BIGINT       NOT NULL COMMENT '患者 ID',
    visit_id        BIGINT       COMMENT '就诊记录 ID',
    doctor_id       BIGINT       NOT NULL COMMENT '开方医生 ID',
    formula_id      BIGINT       COMMENT '基于方剂 ID（若从方剂开方）',
    diagnosis       VARCHAR(200) COMMENT '诊断，如：风寒感冒',
    syndrome        VARCHAR(200) COMMENT '证型，如：风寒束表证',
    notes           TEXT         COMMENT '医嘱备注：用法用量、煎服方法、忌口等',
    status          VARCHAR(20)  DEFAULT 'draft' COMMENT '状态：draft-草稿，pending-待审，approved-通过，rejected-驳回',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (patient_id) REFERENCES patient(id),
    FOREIGN KEY (visit_id)   REFERENCES visit_record(id),
    FOREIGN KEY (doctor_id)  REFERENCES sys_user(id),
    FOREIGN KEY (formula_id) REFERENCES formula(id),
    INDEX idx_patient (patient_id),
    INDEX idx_status (status)
) COMMENT '处方表';
```

```sql
    input_data      TEXT         COMMENT '输入数据（JSON）',

    output_data     TEXT         COMMENT '输出结果（JSON）',
    duration_ms     INT          COMMENT '耗时（毫秒）',
    status          VARCHAR(20)  DEFAULT 'success' COMMENT '状态：success/failed',
    error_message   TEXT         COMMENT '错误信息',
    created_at      DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id)  REFERENCES sys_user(id),
    FOREIGN KEY (visit_id) REFERENCES visit_record(id),
    INDEX idx_visit (visit_id, created_at)
) COMMENT 'Agent 工作流日志表（记录 AI 诊断流程）';
```

### 数据导入说明

Excel 中同一方剂有多行（每行对应一味药材），导入时需：

1. 按 `方剂名` 去重，将方剂信息（名称、分类、功能、主治、修改、来源）导入 `formula` 表
2. 每行药材信息导入 `formula_herb` 表，关联对应的 `formula_id` 和 `herb_id`
3. 药材名去重后导入 `herb` 表（需补充药性、药味、归经、剂量范围等信息）

## 七、数据统计查询说明

`Statistics.vue` 中的统计数据不需要单独的表，通过聚合查询获得：

| 统计项 | 查询方式 |
|--------|----------|
| 月度就诊量 | `SELECT MONTH(visit_date), COUNT(*) FROM visit_record GROUP BY MONTH(visit_date)` |
| 月度开方量 | `SELECT MONTH(created_at), COUNT(*) FROM prescription GROUP BY MONTH(created_at)` |
| 月度审方量 | `SELECT MONTH(created_at), COUNT(*) FROM prescription_review GROUP BY MONTH(created_at)` |
| 证型分布 | `SELECT pattern_name, COUNT(*) FROM diagnosis_result GROUP BY pattern_name` |
| 药材使用排名 | `SELECT herb_name, COUNT(*) FROM prescription_item GROUP BY herb_id ORDER BY COUNT(*) DESC` |
| 诊断排名 | `SELECT diagnosis, COUNT(*) FROM emr GROUP BY diagnosis ORDER BY COUNT(*) DESC` |
| 疗效统计 | `SELECT treatment_result, COUNT(*) FROM emr WHERE treatment_result IS NOT NULL GROUP BY treatment_result` |
| 疗效月度趋势 | `SELECT MONTH(visit_date), treatment_result, COUNT(*) FROM emr GROUP BY MONTH(visit_date), treatment_result` |
| 审方通过率 | `SELECT DATE(created_at), SUM(CASE WHEN review_status='approved' THEN 1 ELSE 0 END)/COUNT(*) FROM prescription_review GROUP BY DATE(created_at)` |
