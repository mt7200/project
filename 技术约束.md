## 二、项目整体架构

### 2.1 低代码开发模式

本项目基于 **SunSheen Hearken 低代码平台** 开发，采用"场景单元(Scene Unit)"开发模式：

```
scenes/                          # 场景目录
├── sysman/                      # 系统管理场景
│   ├── LoginManagement/         # 登录管理场景单元
│   │   ├── CODE.m/              # Java 后端业务逻辑组件
│   │   │   └── component/       # BixComponent 组件
│   │   └── .sceneunit           # 场景单元定义
│   ├── LogQuery/                # 日志查询场景单元
│   │   ├── CODE.m/              # Java 代码
│   │   ├── CONFIG.m/            # 配置
│   │   ├── DATA.m/              # 数据模型/命名空间
│   │   ├── PAGE.m/              # Vue 页面组件
│   │   └── SERVICE.m/           # 服务接口
│   └── .scene                   # 场景定义
└── ...tcm业务场景.../
    ├── PatientEntry/            # 患者入院登记场景单元
    ├── Workspace/               # 四诊信息录入
    ├── DiagnosisFlow/           # 辨证诊断流程
    ├── PrescriptionFlow/        # 开方流程
    ├── PrescriptionReview/      # 处方审核
    ├── EMRViewEdit/             # 电子病历
    └── Statistics/              # 数据统计
```

每个场景单元包含以下元数据目录：

| 目录 | 用途 | 内容 |
|------|------|------|
| `CODE.m/` | Java 业务组件 | 基于 `@BixComponent` 注解的 Spring Bean，通过 `@WebRemote` 暴露远程接口 |
| `CONFIG.m/` | 场景单元配置 | JSON 格式的配置元数据 |
| `DATA.m/*.ns/` | 数据命名空间 | `.dix` 数据库映射文件（SQL 模板）或 `.dm` XML 映射文件 |
| `PAGE.m/` | 前端页面 | Vue 单文件组件（SFC），可引用 `hearken-web-jfui` 低码组件 |
| `SERVICE.m/` | 服务层 | 服务接口定义 |

### 3.3 路由定义

```typescript
// router/index.ts
const router = createRouter({
  history: createWebHashHistory(),  // Hash 模式
  routes: [...staticRoutes, ...devRoutes]
})

// staticRoutes.ts 静态路由（无需认证）
// /auth/login           -> Login.vue（系统管理登录页）
// /auth/register        -> 注册页
// /auth/forget-password -> 找回密码
// /403 / 404 / 500      -> 异常页

// asyncRoutes.ts 动态路由（从后端菜单接口 /api/auth/getUserMenu 加载）
// 菜单结构由后端动态下发，前端根据菜单递归生成路由
```

### 3.4 Axios HTTP 封装

- 基础 URL：通过 `VITE_API_URL` 环境变量配置
- 自动添加 Bearer Token（从 `useUserStore()` 获取）
- 支持 CAS 认证重定向（440 状态码处理）
- 401 自动登出（带 3s 防抖）
- 请求失败自动重试（可配置）
- 统一错误消息提示（可通过 `showErrorMessage` 控制）

```java
    public Object run(Map param) {
        // 调用 DATA.m 中的 SQL 映射
        // 返回查询结果
    }
}
```

### 4.5 SQL 映射文件 (.dm / .dix)

使用 MyBatis 风格的 SQL 模板语法，条件查询通过 `<if test="...">` 标签实现：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<sqlMap namespace="patient">
    <database id="patient" description="">
        <![CDATA[
            -- 查询患者列表（分页）
            SELECT id, name, gender, age, phone, status
            FROM patient
            WHERE 1=1
            <if test="name != null and name != ''">
                AND name LIKE CONCAT('%', #{name}, '%')
            </if>
            ORDER BY created_at DESC
            LIMIT #{offset}, #{pageSize}

            -- 新增患者
            INSERT INTO patient (name, gender, age, phone, ...)
            VALUES (#{name}, #{gender}, #{age}, #{phone}, ...)
        ]]>
    </database>
</sqlMap>
```

## 八、业务逻辑约束

### 8.1 处方剂量校验规则

| 校验项 | 规则 | 风险等级 |
|--------|------|----------|
| 最小剂量 | `dosage ≥ herb.min_dosage` | warning |
| 最大剂量 | `dosage ≤ herb.max_dosage` | high |
| 超量比例 | `dosage > max_dosage * 1.5` | critical |

### 8.2 配伍禁忌检测规则

十八反/十九畏检测，在 Java 服务组件中实现（位于 `prescription.ns` 场景单元的 `CODE.m/` 中）。

### 8.3 审核流程状态机

```
draft → pending → approved/rejected
         ↓
       (驳回后可重新提交 pending)
```

### 8.4 证型编码规范

```
pattern_code 格式：B + 两位大类 + 两位小类
例如："B01.02" = 肝系病证 - 第 2 号子项
```
